CSSで好きな記述

本当はカスケードレイヤー@layerについて考えたかったんだけど、CSSの色々を書いてたら「あーこの記述すき！」というのが頭から離れなくなったので、それについて書き殴ることにする。

calc((100% - 40px) / 3)

  li {
    width: calc((100% - 40px) / 3);
  }

ズバリこれ！これはトップのカード型の枠の幅。
何が好きかって、3で割れないものでもCSSでそれなりにしてくれるという曖昧さ。
昔のデザイナーさんてPCの事だけ考えれば良かったけど、今やデバイスが増えて画面サイズもさまざま。でもCSSが3で割ってくれちゃうというCSSへの依存度を高めさせてくれる曖昧さがたまらない。 いいんですCSSがそう指示してブラウザが再現してくれるんだから。
いや、細かく指定があれば指定しますけどね。
というスタンスでも割と許してもらえるのがこの記述。

justify-content: space-between;

  ul {
    display: flex;
    justify-content: space-between;
    flex-wrap: wrap;
  }

そしてこれ！これもトップのカード型flex。
やはり割り切れないかもしれないものをCSSがカード同士の余白を勝手に割ってくれちゃうやつ。理由は先出のやつと同じ理由。
デザイン的な意図でも無い限り、もうCSSとかブラウザに任せちゃいましょ。という人任せ感を持てる点は、これまでCSSを散々書いてきた中で気が抜けるひと時だったりする。

このあたりのCSSならスマホのブラウザのバージョンを気にせずに使えるところも大きなポイント。
スマホのブラウザはアプデしててもそれ以前にOSが古いどころか機種が古い人も世の中には沢山いる。そうなると「最新バージョンのブラウザで〜」というのも理解されない場合も多いのでサービスによっては機会損失を増やしかねないけど、この2つはほぼセーフなので安心できる。だから好き。
